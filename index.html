Comment utiliser GitHub - Tutoriel complet

GitHub est une plateforme puissante pour le contrôle de version, l'hébergement de code et la collaboration entre développeurs. Voici un guide détaillé pour vous aider à démarrer avec GitHub.

1. Créer un compte GitHub

La première étape consiste à créer un compte sur GitHub. Rendez-vous sur GitHub Join et suivez les instructions.

2. Créer un nouveau repository

Un repository (repo) est un espace où vous pouvez stocker vos projets de code. Voici comment créer un repository sur GitHub :

  1. Connectez-vous à votre compte GitHub.
  2. Dans la barre de navigation supérieure, cliquez sur le bouton New.
  3. Donnez un nom à votre repository (par exemple, "mon-projet").
  4. Choisissez si votre repository sera public ou privé.
  5. Optionnellement, ajoutez un fichier README.md pour décrire votre projet.
  6. Cliquez sur Create repository pour finaliser la création.

3. Cloner un repository

Cloner un repository vous permet de copier un projet GitHub sur votre machine locale afin d'y apporter des modifications. Voici comment faire :

  1. Rendez-vous sur la page du repository sur GitHub.
  2. Cliquez sur le bouton Code et copiez l'URL du repository (HTTPS ou SSH).
  3. Dans votre terminal, naviguez vers le répertoire où vous souhaitez cloner le projet.
  4. Exécutez la commande suivante :
    5. git clone https://github.com/votre-utilisateur/mon-projet.git
  5. Entrez votre mot de passe si nécessaire, et Git va télécharger le repository sur votre machine locale.

4. Effectuer des modifications et créer des commits

Une fois que vous avez cloné le repository, vous pouvez apporter des modifications à votre code. Après avoir modifié des fichiers, vous devez les enregistrer dans Git avec un commit.

  1. Ouvrez un terminal et allez dans le répertoire du projet cloné.
  2. Modifiez vos fichiers à l'aide de votre éditeur de code préféré.
  3. Une fois vos modifications terminées, ajoutez-les à l'index avec la commande :
    4. git add .
  4. Ensuite, créez un commit avec la commande :
    5. git commit -m "Votre message de commit"
  5. Le message de commit doit décrire brièvement les changements que vous avez apportés.

5. Pousser les modifications sur GitHub

Une fois que vous avez créé un commit, vous devez pousser vos modifications vers GitHub pour les partager avec d'autres développeurs. Utilisez la commande suivante :

git push origin main

Assurez-vous de remplacer "main" par la branche sur laquelle vous travaillez, si ce n'est pas la branche principale.

Après avoir poussé les modifications, vous pourrez voir les mises à jour sur la page GitHub de votre repository.

6. Collaborer avec les autres via les Pull Requests

Les Pull Requests (PR) sont utilisées pour proposer des modifications à un repository. Si vous contribuez à un projet open-source, c'est ainsi que vous soumettez vos modifications pour qu'elles soient examinées et fusionnées dans le code principal.

  1. Faites un fork du repository auquel vous voulez contribuer en cliquant sur le bouton Fork sur la page du repository.
  2. Clonez le repository forké sur votre machine.
  3. Apportez les modifications nécessaires et créez un commit comme expliqué précédemment.
  4. Poussez vos modifications sur votre fork.
  5. Sur la page GitHub de votre fork, cliquez sur le bouton Compare & pull request pour proposer vos modifications au repository d'origine.
  6. Rédigez un message de PR et soumettez-le pour qu'il soit examiné par les mainteneurs du projet.

7. Gérer les branches

Les branches permettent de travailler sur des fonctionnalités ou des correctifs sans affecter la branche principale (souvent appelée "main" ou "master"). Voici quelques commandes de base pour gérer les branches :

  • Créer une nouvelle branche : 
    git checkout -b nouvelle-branche
  • Basculer entre les branches : 
    git checkout nom-de-la-branche
  • Fusionner une branche dans la branche principale : 
    git merge nom-de-la-branche
  • Supprimer une branche locale : 
    git branch -d nom-de-la-branche
